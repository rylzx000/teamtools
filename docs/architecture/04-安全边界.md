# 安全边界

## 基本原则

- 用户只能访问自己的任务。
- 管理员可以查看全部任务。
- 团队公共模型 key 不写入前端代码。
- 用户个人模型 key 首版可由用户在页面临时填写并保存在浏览器本地。
- 不修改全局环境变量。
- 上传文件、任务过程文件和结果文件只能进入受控数据目录。
- 后端生成 AI 请求包时不读取无关任务目录、平台敏感配置、环境变量或系统敏感路径。

## 目录边界

运行数据目录：

```text
D:\project\teamtools\data
```

Linux 服务器示例：

```text
/data/teamtools/data
```

主要子目录：

- `data/config/`：配置文件，普通用户不可访问。
- `data/modules/`：模块资料、样例和运行资源。
- `data/tasks/`：任务输入、过程文件和输出文件。
- `logs/` 或 `TEAMTOOLS_LOG_DIR`：平台运行日志。

FPA 模块生成 AI 请求包时，只允许读取：

- 当前任务目录。
- 当前任务选择的系统资料目录。
- FPA 资源目录。

AI 请求包不得包含：

- 服务器绝对路径。
- 平台敏感配置。
- 模型 key。
- Cookie。
- 环境变量。
- 其他用户任务内容。

## 记录边界

数据库只保存运行所需的最小信息：

- 用户
- 任务
- 状态
- 文件相对路径
- 日志摘要
- 事件记录

完整日志和大体积排查文件保存在文件系统，且仅管理员可查看。
