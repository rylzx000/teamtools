# 编码与文本文件规范

## 目标

统一项目文本文件编码，避免 Windows、Linux、Python、PowerShell、Node.js 之间出现中文乱码或测试失败。

## 统一规则

- 项目文本文件统一使用 UTF-8。
- Markdown、Python、JSON、YAML、TypeScript、JavaScript、HTML、CSS、PowerShell、TXT 文件必须是 UTF-8。
- 面向 Excel 直接打开的 `.csv` 文件优先使用 UTF-8 with BOM。
- 不允许在同一个项目中混用 GBK、UTF-16、UTF-8 with BOM、UTF-8 无 BOM，除 `.csv` 特例外。
- 新增文件优先通过支持 UTF-8 的编辑器创建，并遵守根目录 `.editorconfig`。

## 编码写入要求

### Python

读写文本文件必须显式指定编码：

```python
Path("file.md").read_text(encoding="utf-8")
Path("file.md").write_text(content, encoding="utf-8")
```

写给 Excel 直接打开的 CSV：

```python
Path("result.csv").write_text(content, encoding="utf-8-sig")
```

捕获子进程中文输出：

```python
subprocess.run(
    args,
    capture_output=True,
    text=True,
    encoding="utf-8",
    errors="replace",
)
```

### PowerShell

写中文文本必须显式指定编码：

```powershell
Set-Content -LiteralPath .\file.md -Value $text -Encoding utf8
Add-Content -LiteralPath .\file.md -Value $text -Encoding utf8
```

不要使用未指定编码的 `>`、`Out-File`、`Set-Content` 写中文文件。

### Node.js

读写文本文件必须显式指定编码：

```ts
await fs.readFile(path, "utf8");
await fs.writeFile(path, content, "utf8");
```

## 检查命令

提交或交付前运行：

```powershell
.\scripts\check-encoding.ps1
```

该脚本检查常见文本文件是否可按 UTF-8 解码，并跳过二进制文件、缓存目录和运行产物目录。

## 处理乱码文件

发现乱码时不要直接覆盖原文件，应先确认原始编码和来源：

1. 确认文件是否本来应为文本文件。
2. 确认是否来自 Excel、第三方系统或人工复制。
3. 明确原编码后再转换为 UTF-8。
4. 转换后运行编码检查和相关测试。
