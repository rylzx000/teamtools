# 服务器部署

## 部署原则

- 单机部署。
- 推荐使用 Docker Compose。
- 首版不使用 Nginx，直接通过应用端口访问。
- 首版数据库使用 SQLite，数据库文件挂载到宿主机目录。
- 前端静态文件由 FastAPI 提供。
- 运行数据放在工程根目录下的 `data/`。
- 日志放在工程根目录下的 `logs/`。
- 不在服务器全局 Python 环境中安装项目依赖。

## 推荐部署形态

首版推荐一个应用容器：

```text

  -> API
```

访问方式：

```text

```

首版不需要独立后台任务容器，不需要 Redis，不需要 PostgreSQL。

## Docker Compose 文件

工程根目录已提供：

```text
/data/teamtools/Dockerfile
/data/teamtools/docker-compose.yml
/data/teamtools/.dockerignore
/data/teamtools/.env.example
```

部署时复制环境变量样例：

```bash
cp .env.example .env
```

说明：

- `./data:/app/data` 用于保存上传文件、任务结果、SQLite 数据库和运行产物。
- `./logs:/app/logs` 用于保存应用日志。
- 容器重建不会删除 `data/` 和 `logs/`。
- `TEAMTOOLS_PORT` 控制宿主机访问端口，默认 `8000`。
- `.env` 只保存非敏感或部署必要配置；不要把用户模型 API Key 写入公共前端代码。

## 模型直连前置检查

首版由用户浏览器调用 DeepSeek 等外部模型。部署前必须在目标内网访问环境验证：

```text
浏览器页面 -> 模型 API
```

检查项：

- CORS 不拦截浏览器请求。
- API Key 能按模型入口要求传入。
- 浏览器能拿到完整响应。
- 网络失败、鉴权失败、限流失败时，前端能拿到可脱敏展示的错误信息。

当前已通过 DeepSeek 浏览器直连最小测试，确认 API 调用流程可连通。正式部署前使用有效账号再验证一次成功响应即可。

如果浏览器不能直连模型 API，不要继续按当前前端直连方案开发，应先确认企业代理入口或改为后端模型代理方案。

## Dockerfile 原则

当前 Dockerfile 使用两阶段构建：

- Node.js 阶段构建 `frontend/dist`。
- Python 阶段安装后端依赖，复制后端代码、FPA 脚本、模块配置和前端构建产物。
- 运行时通过 `/app/data` 和 `/app/logs` 挂载宿主机数据。
- 容器启动 FastAPI 并监听 `0.0.0.0:8000`。

容器启动命令示例：

```text
uvicorn app.main:app --app-dir /app/backend --host 0.0.0.0 --port 8000
```

注意：`./data` 会挂载覆盖容器内 `/app/data`。因此服务器源码目录中的 `data/config/` 和 `data/modules/` 必须随项目一起保留，不要只创建空 `data/` 目录后覆盖掉模块资源。

## 首次部署步骤

1. 确认服务器已安装 Docker 和 Docker Compose。
2. 将项目源码放到 `/data/teamtools`。
3. 在 `/data/teamtools` 下创建 `data/` 和 `logs/`。
4. 复制 `.env.example` 为 `.env`，按需调整端口。
5. 构建并启动容器。
6. 访问健康检查接口。
7. 创建管理员账号和普通用户账号。
8. 使用浏览器访问平台页面。

示例命令：

```bash
cd /data/teamtools
mkdir -p data logs
cp .env.example .env
docker compose build
docker compose up -d
docker compose ps
```

健康检查：

```text
http://服务器IP:8000/api/health
```

本机服务器内检查：

```bash
curl http://127.0.0.1:8000/api/health
```

## 账号初始化

正式部署默认不启用开发种子账号，`.env` 中应保持：

```bash
TEAMTOOLS_SEED_DEV_USERS=false
```

生产和本地开发均按生产初始化用户口径准备账号：用户名使用拼音，初始化密码来源为手机号或等价初始密码来源；任毅为管理员。管理员可以把用户密码重置为初始化密码，但页面和普通接口不展示手机号或初始化密码来源。

首次部署后可以逐个创建账号：

```bash
docker compose exec teamtools python scripts/create-user.py --username renyi --display-name 任毅 --role admin
docker compose exec teamtools python scripts/create-user.py --username zhangsan --display-name 张三 --role user
```

如果账号已存在并需要改密码、改角色或改展示名，增加 `--update`：

```bash
docker compose exec teamtools python scripts/create-user.py --username zhangsan --display-name 张三 --role user --default-system-code claimcar --update
```

脚本未传 `--password` 时会在命令行提示输入密码，避免密码直接出现在命令历史中。

如需单独指定管理员重置密码使用的初始化密码来源，可传入 `--initial-password-seed`；不传时默认使用本次设置的登录密码作为初始化密码来源。

如果已经准备好初始化账号 Excel，推荐批量导入。Excel 默认格式：

| 第 1 列 | 第 2 列 | 第 3 列 | 第 4 列，可选 | 第 5 列，可选 |
|---|---|---|---|---|
| 姓名 | 用户名 | 初始化密码 | 角色 | 默认系统编码 |

当前初始化账号 Excel 没有表头时，直接执行预览：

```bash
docker compose exec teamtools python scripts/import-users-from-excel.py /tmp/初始化账户.xlsx --admin-username renyi --dry-run
```

确认用户名、姓名和角色无误后导入：

```bash
docker compose exec teamtools python scripts/import-users-from-excel.py /tmp/初始化账户.xlsx --admin-username renyi
```

如果 Excel 第一行是表头，增加 `--has-header`。如果账号已存在并需要覆盖密码、角色、展示名和默认系统，增加 `--update`。

注意：初始化 Excel 中包含手机号形式的初始密码，属于敏感材料；不要提交到 Git，不要放进镜像，导入后应从服务器临时目录删除或妥善保管。

## 生产 FPA 系统范围

上线首版仅对用户开放两个 FPA 系统：

| 系统编码 | 系统名称 |
|---|---|
| `claimcar` | 车险理赔核心系统 |
| `claimoth` | 非车险理赔核心系统 |

提交页和后端配置接口只返回以上两个系统。历史资料目录如 `onlineclaim`、`clqp` 暂不删除，但首版不展示、不初始化、不默认选择。用户没有默认系统时，提交页不自动选择系统，必须由用户明确选择后才能提交。

## 数据库选择

MVP 使用 SQLite：

```text
/data/teamtools/data/teamtools.db
```

适用原因：

- 首批用户约 10 人。
- 并发不高。
- 任务主要是上传、回传 AI 结果、生成 Excel 和下载。
- 部署和备份简单。

后续如推广到部门、并发增加或需要更强审计能力，可切换 PostgreSQL。

PostgreSQL 后续可以加入 `docker-compose.yml`，不需要手工安装到服务器系统中。

## 文件持久化

以下目录必须保留在宿主机：

```text
/data/teamtools/data
/data/teamtools/logs
```

`data/` 建议包含：

```text
data/
├─ teamtools.db
├─ modules/
└─ tasks/
   └─ fpa/
      └─ {task_id}/
         ├─ input/
         ├─ ai/
         ├─ output/
         └─ runtime/
```

`tasks/` 保存任务输入、AI 请求包、AI 调用结果、生成过程和 Excel 输出。

不要把上传文件、AI 产物、Excel 输出、SQLite 数据库只保存在容器内部。

## 对外访问

首版直接端口访问：

```text
http://服务器IP:8000
```

暂不配置 Nginx。

如后续需要统一域名、HTTPS、访问日志、反向代理或多服务路由，再引入 Nginx 或网关。

## 安全与权限

建议：

- `root` 负责安装 Docker、创建目录和部署。
- 应用在容器内运行。
- 宿主机只暴露必要端口。
- `data/` 和 `logs/` 目录只给部署相关用户和 Docker 进程访问。
- 不把模型 API Key 写死到前端代码。
- 用户自己的模型 API Key 首版保存在浏览器本地。
- `.env` 不提交到代码仓库。

## 备用：无 Docker 直接部署

如果 Docker 不可用，可以使用直接部署作为备用方案。

直接部署需要服务器安装：

- Python 3.11+
- Node.js 20+，仅服务器构建前端时需要
- systemd

直接部署运行方式：

```text
FastAPI 由 systemd 管理
前端静态文件由 FastAPI 托管
SQLite 文件放在 /data/teamtools/data/teamtools.db
```

服务示例：

```text
teamtools-api.service
```

当前推荐仍是 Docker Compose；直接部署只作为 Docker 不可用时的备选方案。
