# FPA 已确认需求

## 1. 模块目标

FPA 模块用于支持团队成员在内网网页平台自助提交需求说明，后端生成 AI 请求包，前端调用外部模型，后端校验 AI 结构化结果并生成可下载的 Excel 结果。

首版只实现 FPA 工作量评估模块。核心系统运维处理模块后续再单独设计。

首版用户主流程固定为：

```text
提交评估 -> 等待AI调用 -> 查看任务状态 -> 完成后下载 Excel -> 如有需要复制并重新生成
```

首版只面向单个需求的一次评估。每次提交对应一个需求、一个系统、一个任务和一个 Excel 结果。

首版可用性验收不要求人工抽查评估准确性，只验证平台流程、文件生成、结果下载和关键规则是否可用。

## 2. 用户与权限

- 每个用户单独登录。
- 普通用户只能查看和操作自己提交的任务。
- 管理员可以查看全部用户提交的任务。
- 历史任务默认保留，不提供删除任务记录功能。

## 3. 任务输入

用户提交 FPA 任务时，首版支持以下输入：

- 系统选择，只能选择一个系统。
- 需求名称，可为空。
- 粘贴文本或 Markdown 内容。
- 上传单个 `.md` 文件。
- 粘贴内容和上传 `.md` 文件可以同时提供。
- 目标人天，可选，单位为人天。
- 规模计数时机，可选，默认 `估算早期`。

输入校验规则：

- 粘贴文本和上传 `.md` 至少提供一项。
- 粘贴文本有效内容不超过 2 万字符。
- 上传 `.md` 文件有效内容不超过 2 万字符。
- 上传 `.md` 文件大小不超过 256KB。
- 目标人天为空或大于 0。
- 目标人天最多保留 1 位小数。
- 目标人天不设置业务上限。
- 规模计数时机首版页面提供 `估算早期`、`估算中期`、`估算晚期`。

需求名称为空时，按以下规则生成名称：

- 优先由 AI 或脚本总结短名称。
- 仍无法总结时，使用 `FPA工作量评估-YYYYMMDD-HHmm`。

目标人天用于 AI 参考校准、结果对照和风险提示，不用于阻塞任务。明显异常的目标人天只提示风险，不阻止提交。AI 可以在 FPA 框架内参考目标人天做可解释校准，但不得为了目标人天反推、凑值或无依据新增功能点。

当用户同时提供粘贴文本和上传 `.md` 时，后台按“粘贴文本在前、上传文件在后”生成合并输入，供后端生成 AI 请求包。

## 4. 系统资料

系统资料由管理员提前放到服务器指定目录。普通用户只在页面上选择系统，不需要关心资料目录是否完整。

系统选择规则：

- 每个任务只能选择一个系统。
- 支持“其他系统”。
- 选择“其他系统”时进入无资料模式。
- 已配置系统的资料目录或精简知识包缺失时，任务失败并提示系统资料配置错误，不自动降级为无资料模式。
- 后端按 FPA 资源包规则使用系统资料，生成必要资料摘要或无资料模式说明。
- 资料路由、资料检索和知识使用策略由系统资料或 FPA 规约内部承载，不在平台代码中硬编码。
- MVP 不维护系统资料版本号，也不在任务中记录资料版本。
- 系统资料由管理员在服务器文件中手动维护。

## 5. 处理方式

首版不让大模型直接生成 Excel。处理链路如下：

1. 后台把用户输入保存为任务文件。
2. 后端读取 FPA 资源包、系统资料配置和页面参数，生成 AI 请求包。
3. 前端获取 AI 请求包并调用外部模型。
4. 前端把模型原始响应、AI 分析说明和结构化 JSON 回传后端。
5. 后端校验结构化 JSON，生成 `FPA生成过程.json`。
6. 后台脚本根据 JSON 和 Excel 模板生成结果文件。

这样可以降低模型耗时和 token 消耗，也便于排查、复现和重新生成。

JSON 字段设计原则：

- 以现有 Excel 模板需要填充的字段为准。
- 同时保留 Excel 填充值和 AI 解释文本。
- 脚本只使用结构化填充值生成 Excel。
- AI 解释文本用于页面查看和人工理解。
- 字段级细节在接入模板和脚本时进一步设计。

## 6. AI 调用或 JSON 校验失败

如果前端调用模型失败：

- 前端回传脱敏后的错误信息。
- 后端保存错误详情，任务状态改为失败。
- 普通用户看到失败阶段、具体原因和建议操作。
- 管理员可以查看模型调用错误摘要和任务日志。

如果后端校验 AI 结构化 JSON 失败：

- 任务失败，不由后端自动调用模型修复。
- 失败详情需要记录失败阶段、具体原因和建议操作。
- 用户可复制原输入重新生成，或前端按页面规则重新调用模型后回传新结果。

## 7. 功能点条目数量

首版不设置固定功能点条目上限。

生成 Excel 时：

- 所有条目生成在同一个 Excel 文件中。
- 不拆分多个 Excel。
- 条目超过模板默认行数时，通过脚本动态增加行。
- 条目数量明显过多时，可在页面和 `AI分析.md` 中提示风险。

## 8. Excel 模板与结果计算

Excel 必须按现有模板生成。

模板处理规则：

- 保留模板中的工作表。
- 保留模板中的格式。
- 保留模板中的公式。
- 保留模板中的参数区。
- 脚本只填充模板约定的输入区域和明细行。
- 脚本不主动覆盖公式单元格。
- 人工后续可以在 Excel 中继续调整参数。

后台计算规则：

- 不引入 LibreOffice。
- 不引入 Microsoft Office 服务端依赖。
- Excel 文件中的公式保留给用户打开后自动重算。
- 页面展示的关键结果由脚本按模板公式口径同步计算。
- `结果中值` 和 `是否命中目标` 来自脚本计算结果。
- 脚本计算逻辑必须与 Excel 模板公式保持一致。
- 模板文件由管理员手动替换。
- MVP 不维护模板版本号，也不在任务中记录模板版本。

## 9. 任务状态

任务状态使用以下集合：

- 等待AI调用
- 结果校验中
- 生成Excel中
- 已完成
- 失败
- 已取消

可取消状态：

- 等待AI调用
- 结果校验中
- 生成Excel中

不可取消状态：

- 已完成
- 失败
- 已取消

## 10. 取消任务

用户可以取消仍在执行中的任务。

取消行为：

- 任务先标记 `cancel_requested`。
- 后端尽量停止正在执行的校验或生成脚本。
- 如果前端正在调用模型，前端负责中止请求或提示用户关闭等待。
- 状态改为已取消。
- 记录取消时间和取消人。
- 清理临时文件。
- 保留任务记录和取消日志摘要。

## 11. 失败任务

失败任务需要保留排查信息。

普通用户可见信息：

- 失败阶段。
- 脱敏后的具体原因。
- 建议操作。

管理员额外可见信息：

- 完整任务日志。
- 异常堆栈。
- 工作目录。
- 模型请求和响应摘要。

普通用户不能看到模型 key、服务器敏感路径、完整源码路径和环境变量。

失败任务支持重新运行。重新运行时基于原输入新建一个新任务，原失败任务保留，不覆盖原状态和排查文件。

## 12. 成功任务结果

成功任务保留以下结果：

- Excel 文件，用户可下载。
- `AI分析.md`，页面可查看、可复制，不提供下载。
- `FPA生成过程.json`，页面可查看、可复制，不提供下载。

任务详情页需要展示轻量质量提示，不做评分。

提示项包括：

- 是否无资料模式。
- 是否目标人天异常。
- 是否功能点条目过多。
- 是否发生过 AI 调用失败或 JSON 校验失败。
- 当前资源文件是否可读取。

质量提示只提示风险，不阻止用户下载和使用结果。

Excel 文件命名规则：

- 有需求名称：`需求名称-FPA工作量评估.xlsx`
- 无明确名称但可以总结：`短需求名称-FPA工作量评估.xlsx`
- 无法总结：`FPA工作量评估-YYYYMMDD-HHmm.xlsx`

重复同名任务不覆盖旧结果，每次提交都生成新的任务和新文件。

## 13. 历史任务

历史任务列表需要保留。

普通用户列表字段：

- 任务名称
- 系统
- 状态
- 提交时间
- 完成时间
- 目标人天
- 结果中值
- 是否命中目标
- 操作

管理员列表额外字段：

- 提交人
- 失败阶段

任务详情页需要展示：

- 是否无资料模式
- 耗时
- 重试次数
- 详细错误
- `AI分析.md` 预览
- `FPA生成过程.json` 预览
- Excel 下载入口

## 14. 复制并重新生成

支持基于历史任务复制并重新生成。

规则：

- 不编辑原任务。
- 不做版本管理。
- 点击“复制并重新生成”后，带入旧任务输入内容。
- 旧任务的粘贴文本和上传 `.md` 内容合并回输入框。
- 用户修改后提交新任务。
- 新任务生成新的任务记录和结果文件。

失败任务重新运行也采用同样原则：新建任务，不覆盖原失败任务。

## 15. 模块资产与验收样例

FPA 首版接入的现有资产以以下目录为准：

```text
D:\project\fpa功能点法工作量评估
```

后续实现阶段从该目录确认模板、脚本、规约和示例文件。

首版需要保留示例输入和示例输出作为开发与验收样例，但不放到前端页面。

验收样例用途：

- 验证任务提交、AI 请求包生成、AI 结果回传和结果生成流程是否跑通。
- 验证 AI 结构化 JSON 和 `FPA生成过程.json` 是否能生成。
- 验证 Excel 是否能基于模板生成并下载。
- 验证模板格式、公式和参数区是否保留。
- 验证失败重新运行、历史记录和文件权限是否符合设计。

## 16. AI 请求包边界

AI 请求包由后端生成，前端只负责调用模型。

后端生成请求包时只允许读取以下目录：

- 当前任务目录。
- 当前任务选择的系统资料目录。
- FPA 资源目录。

AI 请求包不得包含服务器绝对路径、其他任务目录、平台敏感配置、模型 key、环境变量或系统敏感路径。

## 17. 暂不实现内容

以下内容首版暂不实现：

- 草稿保存。
- 在线编辑评估结果。
- 人工审批。
- 多人协同复核。
- 批量评估。
- 评估结果对比。
- 页面维护系统资料。
- 模板版本管理页面。
- 系统资料版本管理页面。
- FPA 模块级模型参数配置。
- 管理员统计报表。
- 人工抽查准确性流程。
- PDF、Word、Excel 等非 Markdown 输入解析。
- 多系统同时选择。
- 多个 Excel 拆分输出。
- 用户下载 `AI分析.md` 或 `FPA生成过程.json`。
- 删除历史任务。
- 核心系统运维处理模块。
