# FPA 接口设计

## 1. 设计目标

FPA 首版接口围绕“任务资源”设计，前端通过任务接口完成提交、获取 AI 请求包、回传模型结果、查询、取消、重新运行和 Excel 下载。

接口只服务 FPA 模块，不混合其他模块任务。平台后续可以做统一任务中心，但 FPA 模块内列表只展示 FPA 任务。

## 2. 页面与接口关系

首版 FPA 模块包含 3 个页面：

| 页面 | 主要接口 | 说明 |
|---|---|---|
| 提交评估 | `GET /api/fpa/systems`、`POST /api/fpa/tasks`、`GET /api/fpa/tasks/{id}/ai-request`、`POST /api/fpa/tasks/{id}/ai-result` | 选择系统、输入需求、上传 `.md`、填写目标人天、执行模型调用并回传结果 |
| 任务列表 | `GET /api/fpa/tasks` | 查看 FPA 历史任务，列表只手动刷新 |
| 任务详情 | `GET /api/fpa/tasks/{id}`、取消、重新运行、下载接口 | 查看状态、错误、AI 分析、JSON、下载 Excel |

任务详情页每 2-3 秒轮询一次 `GET /api/fpa/tasks/{id}`。任务列表页不自动刷新，只提供手动刷新按钮。

`AI分析.md` 和 `FPA生成过程.json` 在任务详情页使用折叠区域展示，支持复制，不提供下载。

## 3. 通用约定

### 3.1 权限

- 普通用户只能访问自己的 FPA 任务。
- 管理员可以访问全部 FPA 任务。
- Excel 下载、取消任务、重新运行任务都必须由后端校验权限。
- 前端可以隐藏或置灰按钮，但不能作为权限判断依据。

### 3.2 响应格式

成功响应返回业务对象。

错误响应统一使用：

```json
{
  "error": {
    "code": "string",
    "message": "string",
    "detail": "object"
  }
}
```

普通用户错误信息需要脱敏。管理员可以查看更完整的错误详情。

### 3.3 常用状态码

| 状态码 | 说明 |
|---:|---|
| `200` | 查询、取消、重新运行成功 |
| `201` | 创建任务成功 |
| `400` | 请求参数错误 |
| `401` | 未登录 |
| `403` | 无权限 |
| `404` | 任务或资源不存在 |
| `409` | 当前状态不允许执行该操作 |
| `413` | 上传内容超过限制 |
| `500` | 服务端错误 |

## 4. 接口清单

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/api/fpa/systems` | 查询可选系统 |
| `POST` | `/api/fpa/tasks` | 创建 FPA 任务 |
| `GET` | `/api/fpa/tasks/{id}/ai-request` | 获取后端生成的 AI 请求包 |
| `POST` | `/api/fpa/tasks/{id}/ai-result` | 回传模型成功结果或调用错误 |
| `GET` | `/api/fpa/tasks` | 查询 FPA 任务列表 |
| `GET` | `/api/fpa/tasks/{id}` | 查询 FPA 任务详情 |
| `POST` | `/api/fpa/tasks/{id}/cancel` | 取消任务 |
| `POST` | `/api/fpa/tasks/{id}/rerun` | 基于原输入重新运行 |
| `GET` | `/api/fpa/tasks/{id}/download/excel` | 下载 Excel 结果 |

## 5. 查询可选系统

```http
GET /api/fpa/systems
```

后端读取 `systems.yaml`，只返回 `enabled = true` 的系统。

响应字段：

```json
{
  "systems": [
    {
      "code": "claimcar",
      "name": "车险理赔核心系统",
      "system_type": "核心系统",
      "sort_order": 10
    },
    {
      "code": "claimoth",
      "name": "非车险理赔核心系统",
      "system_type": "核心系统",
      "sort_order": 20
    },
    {
      "code": "onlineclaim",
      "name": "在线理赔服务平台",
      "system_type": "多流程支持系统",
      "sort_order": 30
    },
    {
      "code": "clqp",
      "name": "零配件报价系统",
      "system_type": "专业工具",
      "sort_order": 40
    }
  ]
}
```

说明：

- 不向前端返回 `knowledge_dir`。
- 不向前端返回服务器真实资料路径。
- 排序按 `sort_order` 升序。
- 首批系统编码固定为 `claimcar`、`claimoth`、`onlineclaim`、`clqp`。前端提交和后端匹配必须使用这些编码，不得自行改成 `claim_core`、`online_claim` 或其他别名。

## 6. 创建任务

```http
POST /api/fpa/tasks
```

请求类型建议使用 `multipart/form-data`，同时支持文本和文件上传。

请求字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `system_code` | string | 是 | 用户选择的系统编码 |
| `title` | string | 否 | 需求名称，允许为空 |
| `input_text` | string | 否 | 粘贴输入 |
| `input_file` | file | 否 | 上传 `.md` 文件 |
| `target_person_days` | number | 否 | 目标人天，大于 0，最多 1 位小数 |
| `count_timing` | string | 否 | 规模计数时机，默认 `估算早期` |

校验规则：

- `input_text` 和 `input_file` 至少提供一项。
- 粘贴文本有效内容不超过 2 万字符。
- 上传 `.md` 有效内容不超过 2 万字符。
- 上传 `.md` 文件大小不超过 256KB。
- 每次只能选择一个系统。
- `count_timing` 允许值来自模板基础参数，MVP 页面只提供 `估算早期`、`估算中期`、`估算晚期`。
- 未传 `count_timing` 时，后端使用 `估算早期`。

成功响应：

```json
{
  "task": {
    "id": "task_id",
    "module": "fpa",
    "status": "waiting_ai_call",
    "status_label": "等待AI调用",
    "title": "需求名称",
    "detail_url": "/fpa/tasks/task_id",
    "ai_request_url": "/api/fpa/tasks/task_id/ai-request"
  }
}
```

前端收到成功响应后跳转任务详情页。

## 7. 获取 AI 请求包

```http
GET /api/fpa/tasks/{id}/ai-request
```

前端使用该接口获取后端生成的 AI 请求包，然后由浏览器侧调用外部模型。

后端必须校验：

- 当前用户是否有权限访问该任务。
- 任务状态是否允许获取请求包。
- AI 请求包文件是否存在。

响应字段：

```json
{
  "task": {
    "id": "task_id",
    "status": "waiting_ai_call",
    "status_label": "等待AI调用"
  },
  "ai_request": {
    "provider_hint": "deepseek",
    "model_hint": "deepseek-v4-flash",
    "messages": [
      {
        "role": "system",
        "content": "系统提示词"
      },
      {
        "role": "user",
        "content": "用户需求与输出要求"
      }
    ],
    "response_format": {
      "type": "json_object"
    }
  }
}
```

说明：

- `provider_hint`、`model_hint` 只是默认建议，前端可根据页面配置调整。
- 响应中不得包含服务器绝对路径、密钥、Cookie 或用户无权查看的内部配置。
- 后端记录 `ai_request_fetched` 事件，便于排查任务卡点。

## 8. 回传 AI 结果

```http
POST /api/fpa/tasks/{id}/ai-result
```

前端调用模型结束后，通过该接口回传成功结果或失败信息。

成功回传请求示例：

```json
{
  "success": true,
  "provider": "deepseek",
  "model": "deepseek-v4-flash",
  "raw_response": {},
  "analysis_md": "可选 Markdown 分析说明",
  "structured_json": {
    "items": []
  }
}
```

失败回传请求示例：

```json
{
  "success": false,
  "provider": "deepseek",
  "model": "deepseek-v4-flash",
  "error": {
    "code": "timeout",
    "message": "模型调用超时"
  }
}
```

后端处理规则：

- 前端本地模型调用失败时，允许在浏览器侧重试，不必立即回传失败。
- 用户放弃重试、达到前端重试上限或确认失败后，前端回传 `success = false`，后端保存错误信息，任务状态改为 `failed`。
- `success = true` 时，保存 AI 原始响应和结构化 JSON，任务状态进入 `validating_result`。
- JSON 校验通过后进入 `generating_result` 并生成 Excel。
- JSON 校验或 Excel 生成失败时，任务状态改为 `failed`。

成功响应：

```json
{
  "task": {
    "id": "task_id",
    "status": "generating_result",
    "status_label": "生成结果中"
  }
}
```

## 9. 查询任务列表

```http
GET /api/fpa/tasks
```

任务列表只返回 FPA 任务。

查询参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `page` | integer | 否 | 页码，默认 `1` |
| `page_size` | integer | 否 | 每页数量 |
| `status` | string | 否 | 按英文状态枚举筛选 |

普通用户只能查询自己的任务。管理员可以查询全部任务。

普通用户列表字段：

```json
{
  "items": [
    {
      "id": "task_id",
      "title": "需求名称",
      "system_name": "车险理赔核心系统",
      "status": "completed",
      "status_label": "已完成",

      "finished_at": "2026-06-24T10:03:00+08:00",
      "target_person_days": 10.0,
      "count_timing": "估算早期",
```

## 11. 取消任务

```http
POST /api/fpa/tasks/{id}/cancel
```

允许取消状态：

- `waiting_ai_call`
- `validating_result`
- `generating_result`

后端必须校验：

- 当前用户是否有权限操作该任务。
- 当前任务状态是否允许取消。

成功响应：

```json
{
  "task": {
    "id": "task_id",
    "status": "canceled",
    "status_label": "已取消",
    "cancel_requested": true
  }
}
```

如果状态不允许取消，返回 `409`。

## 12. 重新运行

```http
POST /api/fpa/tasks/{id}/rerun
```

重新运行基于原任务输入新建一个新任务，不覆盖原任务。

适用场景：

- 失败任务重新运行。
- 历史任务复制后重新生成。

后端必须校验：

- 当前用户是否有权限访问原任务。
- 原任务输入文件是否仍可用。

成功响应：

```json
{
  "source_task_id": "old_task_id",
  "task": {
    "id": "new_task_id",
    "module": "fpa",
    "status": "waiting_ai_call",
    "status_label": "等待AI调用",
    "rerun_from_task_id": "old_task_id",
    "detail_url": "/fpa/tasks/new_task_id",
    "ai_request_url": "/api/fpa/tasks/new_task_id/ai-request"
  }
}
```

前端收到成功响应后跳转新任务详情页。

## 13. 下载 Excel

```http
GET /api/fpa/tasks/{id}/download/excel
```

后端必须校验：

- 当前用户是否有权限访问该任务。
- 任务是否已完成。
- Excel 结果文件是否存在。

普通用户只能下载自己的 FPA Excel。管理员可以下载全部 FPA Excel。

不允许通过该接口下载 `AI分析.md`、`FPA生成过程.json`、任务日志或其他过程文件。

## 14. 前端状态控制

前端可以根据详情接口返回字段控制按钮显示：

- `can_cancel`
- `can_rerun`
- `can_download_excel`
- `can_fetch_ai_request`
- `can_submit_ai_result`

这些字段只用于页面展示。最终操作是否允许，必须以后端接口校验结果为准。
