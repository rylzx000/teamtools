# FPA AI 请求包与提示词合成设计

## 1. 设计目标

本文档定义 FPA 模块中 AI 请求包的生成边界、提示词合成规则、前后端职责、落盘文件和错误处理口径。`AI`、`JSON`、`Excel`、`payload`、`Key`、`DeepSeek` 等英文保留，是既有能力、数据格式、办公软件名称、机器可读数据结构、凭据概念和第三方模型服务名称，避免破坏接口约定。

核心目标：

- 用户交互保持简单：上传或粘贴需求，选择系统，提交任务，最终查看摘要并下载 Excel。
- 后端负责生成 AI 请求包，前端不自行拼接完整提示词。
- 前端负责调用外部模型，并把模型响应回传后端。
- 后端负责校验 AI 结果，并通过脚本确定性生成 `FPA生成过程.json` 和 Excel。
- AI 只做业务功能点识别、参考目标校准和结构化判断，不生成 Excel、不输出最终人天、不决定公式。
- `AI评估.md`、结构化 JSON、Excel payload 和过程 JSON 是管理员或后台排查产物，不面向普通用户文件列表开放。

## 2. 整体流程

主流程如下：

```text
用户上传或粘贴需求
-> 用户选择系统和页面参数
-> 后端创建 FPA 任务
-> 后端读取系统配置、系统知识包、提示词模板和 JSON 契约
-> 后端生成 AI 请求包
-> 前端获取 AI 请求包并调用模型
-> 前端回传 AI 原始响应或调用错误
-> 后端提取 AI评估.md 和 AI结构化结果.json
-> 后端校验事实、路由、拆分/合并和 frozen_items
-> 后端从同一组 frozen_items 生成 Excel 脚本输入 payload
-> 脚本生成 FPA生成过程.json 和 Excel
-> 用户查看摘要并下载 Excel
```

关键原则：

- AI 请求包是任务级快照，任务创建后不随资源文件修改自动变化。
- 已生成的 AI 请求包保存到任务目录，便于管理员排查。
- 前端可展示或复制后端生成的请求内容，但普通用户不通过文件列表下载 AI 请求包。
- 模型调用失败、结构化提取失败或 JSON 校验失败时，任务失败并保留排查信息。

## 3. 前后端职责

| 环节 | 前端 | 后端 |
|---|---|---|
| 收集需求 | 上传 Markdown、Word 或粘贴文本 | 保存并归一化用户输入 |
| 选择系统 | 提供系统下拉选项 | 根据 `systems.yaml` 校验 `system_code` |
| 页面参数 | 提供规模计数时机、完整性级别、目标人天 | 保存任务配置 |
| 读取系统资料 | 不直接读文件 | 读取系统知识包和场景拆分字典 |
| 合成提示词 | 不拼接完整提示词 | 读取模板并替换变量 |
| 调用模型 | 使用用户浏览器环境调用模型 | 不直接调用公网模型 |
| 回传结果 | 回传原始响应或调用错误 | 保存、解析、校验、生成结果 |
| 生成 Excel | 不生成 | 调用脚本确定性生成 |

前端可以提供“复制请求内容”的辅助能力，但复制内容必须来自后端生成的请求包，不允许前端临时拼接另一套提示词。

## 4. 输入来源

AI 请求包由后端基于以下输入合成：

| 输入 | 来源 | 是否必需 | 说明 |
|---|---|---:|---|
| 用户需求正文 | 用户上传或粘贴 | 是 | 需求原文，支持粘贴文本、Markdown 和 Word 归一化正文 |
| 需求标题 | 用户输入或文件名 | 否 | 用于任务展示，AI 可辅助总结但不作为唯一来源 |
| `system_code` | 页面选择 | 是 | 当前生产对外仅提供 `claimcar`、`claimoth` |
| 系统知识包 | `systems.yaml` + 知识包文件 | 条件必需 | 有资料模式必需，无资料模式为空 |
| 系统场景字典 | `08-FPA场景拆分字典.md` | 条件必需 | 存在时作为场景路由和命名关键上下文 |
| 页面参数 | 页面配置 | 是 | 规模计数时机、完整性级别、目标人天 |
| 提示词模板 | `data/modules/fpa/profile/skill/prompt_template.md` | 是 | 缺失则任务失败 |
| JSON 契约 | `data/modules/fpa/profile/schema/result.schema.json` | 是 | 用于约束 AI 输出 |
| FPA 规则摘要 | 资源包文档或模板内固定内容 | 是 | 供 AI 判断类别、复用程度、修改类型 |

## 5. 系统资料读取规则

后端根据任务的 `system_code` 读取：

```text
data/config/modules/fpa/systems.yaml
```

再按系统配置拼接知识包路径。

规则：

- `brief_file` 未配置时默认使用 `teamtools-system-brief.md`。
- `knowledge_dir` 为空时进入无资料模式，`no_knowledge_mode = true`。
- `knowledge_dir` 非空但基础资料不存在时，任务失败，记录为系统资料配置错误。
- 若存在 `08-FPA场景拆分字典.md`，请求包必须把它作为关键上下文提供给 AI。
- 系统知识包超过蒸馏规则定义的长度上限时，任务失败或记录管理员可见配置错误，不静默截断。
- 首版默认读取精简知识包，不读取 `source/` 下完整资料。
- 不做片段检索，不把系统资料全文动态喂给模型。

当前生产对外系统编码固定如下：

| system_code | 系统名称 | 知识包目录 |
|---|---|---|
| `claimcar` | 车险理赔核心系统 | `modules/fpa/knowledge/claimcar` |
| `claimoth` | 非车险理赔核心系统 | `modules/fpa/knowledge/claimoth` |

`onlineclaim`、`clqp` 等历史资料目录暂不删除，但不展示、不初始化、不默认选择。

代码、配置和文档不得自行引入 `claim_core`、`online_claim`、`other_system` 等未确认别名。

## 6. 提示词分层

AI 请求包逻辑上分为三层。

### 6.1 固定角色与边界

用于定义 AI 的角色和硬约束：

- 你是 FPA 功能点识别助手。
- 只输出 `<AI评估.md>` 和 `<AI结构化结果.json>` 两个区块。
- 结构化 JSON 必须使用 `schema_version = fpa.ai_contract.v2`。
- `AI评估.md`、Excel payload 和过程 JSON 都只能来自同一组 `frozen_items`。
- `frozen_items[].system` 必须输出系统中文名，不得输出系统编码。
- 不生成 Excel。
- 不输出最终人天、UFP、US、目标命中、模板路径、Excel 单元格、脚本参数或文件路径。
- 可以参考目标人天，但不得为了贴近目标无依据新增、删除、拆碎、合并、重排或重判功能点。

### 6.2 任务上下文

用于描述本次任务：

- 需求标题。
- 用户需求原文。
- 已选系统名称和 `system_code`。
- 是否无资料模式。
- 页面可控参数。
- 目标人天作为“参考目标人天”写入模型正文，用于辅助 FPA 明细校准；目标命中判断仍由脚本完成。

### 6.3 判断资料与输出契约

用于提供判断依据：

- 已选系统的精简知识包。
- `08-FPA场景拆分字典.md`，如果存在。
- FPA 分类和复用判断规则。
- 拆分完整性要求和关键链路复核表。
- `result.schema.json` 对应的输出结构。
- 输出字段说明、枚举值、禁止项和引用规则。

如果模型接口支持 OpenAI 风格 messages，可以拆为 `system` 和 `user`；如果只支持单段文本，则由后端合成为一个完整 Markdown 文本。`OpenAI` 和 `Markdown` 是接口风格和文档格式名称，保留英文。前端实际调用模型时只能选择 `messages` 或 `plain_prompt` 其中一种发送，不得把两种形态、`metadata` 或排查字段一并发送给模型。

## 7. 模板变量

提示词模板建议保留以下变量：

```text
{{ requirement_title }}
{{ input_text }}
{{ system_code }}
{{ system_name }}
{{ knowledge_summary }}
{{ system_scene_dictionary }}
{{ no_knowledge_mode }}
{{ target_person_days }}
{{ project_features }}
{{ schema_text }}
{{ output_rules }}
```

变量说明：

| 变量 | 说明 |
|---|---|
| `requirement_title` | 任务标题，来自用户输入、文件名或后端生成 |
| `input_text` | 用户需求原文 |
| `system_code` | 页面选择的系统编码 |
| `system_name` | 系统显示名称 |
| `knowledge_summary` | 系统知识包正文；无资料模式为空或写明无资料 |
| `system_scene_dictionary` | 系统场景拆分字典；缺失时写明无系统字典模式 |
| `no_knowledge_mode` | 是否无资料模式 |
| `target_person_days` | 参考目标人天，可为空；供 AI 在 FPA 框架内做可解释校准，不要求 AI 输出 |
| `project_features` | 页面可控项目特征，例如规模计数时机和完整性级别 |
| `schema_text` | AI 输出 JSON 契约 |
| `output_rules` | 输出限制、枚举值和禁止项 |

模板变量缺失或替换失败时，任务失败，不使用隐藏默认提示词。

## 8. AI 请求包文件

每个任务目录下保存 AI 相关文件：

```text
data/tasks/fpa/{task_id}/ai/
  AI请求包.json
  AI请求摘要.json
  AI原始响应.json
  AI调用错误.json
  AI结构化结果.json
  AI评估.md
```

文件职责：

| 文件 | 生成方 | 说明 |
|---|---|---|
| `AI请求包.json` | 后端 | 前端调用模型所需的完整请求包 |
| `AI请求摘要.json` | 后端 | 脱敏摘要，便于排查请求构成 |
| `AI原始响应.json` | 前端回传、后端保存 | 模型原始返回 |
| `AI调用错误.json` | 前端回传、后端保存 | 浏览器调用模型失败时保存 |
| `AI结构化结果.json` | 后端 | 从模型响应中解析并通过校验后的标准 JSON |
| `AI评估.md` | AI 回传、后端保存 | 后台评估说明和管理员排查依据 |

普通用户通过专用接口获取 `AI请求包.json` 的可调用内容，不在文件列表中展示该文件，也不提供下载入口。管理员可查看排查文件。

## 9. `AI请求包.json` 建议结构

建议结构：

```json
{
  "provider": "deepseek",
  "model": "deepseek-v4-flash",
  "request_format": "messages",
  "messages": [
    {
      "role": "system",
      "content": "固定角色和硬约束"
    },
    {
      "role": "user",
      "content": "任务上下文、系统资料、场景字典、需求正文和输出契约"
    }
  ],
  "plain_prompt": "当模型入口不支持 messages 时使用的完整文本",
  "generation_config": {
    "temperature": 0.1,
    "top_p": 0.9
  },
  "metadata": {
    "task_id": "任务ID",
    "module": "fpa",
    "system_code": "claimcar",
    "system_name": "车险理赔核心系统",
    "no_knowledge_mode": false,
    "target_person_days": 10.0,
    "template_version": "文件时间戳或摘要",
    "knowledge_file": "modules/fpa/knowledge/claimcar/teamtools-system-brief.md",
    "system_scene_dictionary_file": "modules/fpa/knowledge/claimcar/08-FPA场景拆分字典.md"
  }
}
```

说明：

- `messages` 用于支持标准聊天接口。
- `plain_prompt` 用于网页端只接受单段文本的模型入口。
- 前端调用模型时必须按 `request_format` 二选一发送：`messages` 模式只发送 `messages`，`plain_prompt` 模式只发送 `plain_prompt`。
- `metadata`、`AI请求摘要.json`、任务路径、文件路径和排查字段不得发送给模型。
- `generation_config` 是建议值，实际前端调用时可以按模型 API 约束转换。
- API Key 不写入请求包，不保存到后端。

## 10. 请求摘要

`AI请求摘要.json` 用于管理员排查，不保存完整需求正文和完整系统知识包。

建议结构：

```json
{
  "task_id": "任务ID",
  "system_code": "claimcar",
  "system_name": "车险理赔核心系统",
  "no_knowledge_mode": false,
  "has_system_scene_dictionary": true,
  "input_chars": 12000,
  "knowledge_chars": 9800,
  "schema_chars": 3000,
  "prompt_chars": 26000,
  "template_file": "data/modules/fpa/profile/skill/prompt_template.md",
  "knowledge_file": "data/modules/fpa/knowledge/claimcar/teamtools-system-brief.md",
  "created_at": "2026-07-05T10:00:00+08:00"
}
```

摘要用途：

- 判断请求是否过长。
- 判断是否进入无资料模式或无系统字典模式。
- 判断使用了哪个系统知识包和提示词模板。
- 排查提示词模板、系统资料和场景字典配置问题。

## 11. Token 与长度控制

首版不做复杂检索和自动压缩，但必须做长度保护。`Token` 是模型计量单位专有名词，保留英文。

建议控制项：

| 项目 | 处理规则 |
|---|---|
| 用户需求正文 | 完整写入请求包；如超过系统配置上限，任务失败并提示用户拆分 |
| 系统知识包 | 只读取精简知识包；超出规则上限则任务失败 |
| 系统场景字典 | 存在时作为关键上下文；超出上限应提示管理员维护精简版本 |
| JSON 契约 | 放入必要字段和枚举说明，不放无关实现细节 |
| 提示词模板 | 保留强约束和输出格式，不写长篇背景解释 |
| 总请求长度 | 后端记录 `prompt_chars`；超过配置阈值时任务失败 |

建议首版配置：

```yaml
max_input_chars: 30000
max_knowledge_chars: 12000
max_prompt_chars: 60000
```

上述是字符级保护，不等同于 token 精确计算。

## 12. 浏览器直连模型验收

由于首版由用户浏览器调用 DeepSeek，开发前必须验证目标模型入口是否允许浏览器直连。

验收项：

| 验收项 | 预期结果 |
|---|---|
| CORS | 浏览器页面可以直接调用模型 API，不被跨域策略拦截 |
| 鉴权 | API Key 可以通过浏览器请求头或模型入口要求的方式传入 |
| 响应格式 | 能拿到完整模型响应，且前端可以回传后端 |
| 错误信息 | 网络失败、鉴权失败、限流失败时能拿到可展示的脱敏错误摘要 |

如果 DeepSeek API 不允许浏览器跨域调用，首版主链路无法按当前方式实现，必须二选一：

- 使用允许浏览器调用的模型入口或企业代理入口。
- 改为后端模型代理模式，并重新评估服务器网络、API Key 保存和审计边界。

## 13. AI 输出要求

AI 必须返回可解析的 Markdown 包裹结果，其中包含 `AI评估.md` 和结构化 JSON。

硬约束：

- 结构化 JSON 顶层必须包含 `schema_version`、`requirement_name`、`assessment_context`、`change_facts`、`routing_decisions`、`split_merge_decisions` 和 `frozen_items`。
- `schema_version` 必须是 `fpa.ai_contract.v2`。
- `frozen_items` 必须是非空数组。
- `frozen_items[].system` 必须是当前任务系统中文名，不允许使用系统编码。
- 每条冻结项必须包含功能点类别、复用程度、修改类型、判断依据、事实引用和路由引用。
- `category` 只能是 `ILF`、`EIF`、`EI`、`EO`、`EQ`。
- `reuse` 只能是 `高`、`中`、`低`。
- `change_type` 只能是 `新增`、`修改`、`删除`。
- 命中系统场景字典时，`system_scene_ids` 必须非空，Excel 一级模块、二级模块和计数项名称必须原样使用字典值。
- `ILF` 和 `EIF` 应通过 `linked_process_ids` 关联支撑事务功能；无法关联时必须在备注说明原因，或在 `review_notes` 中绑定该条 `stable_id` / `count_item_name` 并说明资料不足、无法明确、列入复核或不硬补事务功能等原因。
- `EI`、`EO`、`EQ` 如维护或引用数据功能，应通过 `linked_data_ids` 关联对应数据功能。
- 有目标人天时，AI 可以在功能点拆分粒度、复用程度、修改类型上做可解释校准，但不得改变冻结清单边界。
- 不输出汇总功能点、人天、目标命中、模板路径、Excel 路径、单元格坐标或脚本参数。

如果模型额外输出说明文字，后端应优先提取指定区块；无法稳定提取时，任务失败并保存原始响应。

## 14. 错误处理

AI 请求包生成阶段错误：

| 场景 | 处理 |
|---|---|
| `system_code` 不存在 | 任务失败，提示系统配置不存在 |
| `knowledge_dir` 非空但知识包缺失 | 任务失败，提示系统资料配置错误 |
| 提示词模板缺失 | 任务失败，提示资源包配置错误 |
| JSON 契约缺失 | 任务失败，提示资源包配置错误 |
| 输入正文为空 | 任务失败，提示用户补充需求内容 |
| 输入或请求过长 | 任务失败，提示拆分需求或联系管理员调整配置 |

模型调用阶段错误：

| 场景 | 处理 |
|---|---|
| 前端调用模型失败 | 前端本地允许重试；用户放弃或达到重试上限后，前端回传 `AI调用错误.json`，后端标记任务失败 |
| 浏览器网络不可达 | 页面提示检查网络或 API Key |
| API Key 无效 | 页面展示脱敏错误摘要，不上传 API Key |
| 模型返回空内容 | 后端标记任务失败，管理员可查看原始响应 |

结果解析阶段错误：

| 场景 | 处理 |
|---|---|
| 非法 JSON | 任务失败，可由用户重新调用模型 |
| 缺少指定输出区块 | 任务失败，提示模型输出不符合契约 |
| schema 校验失败 | 任务失败，保存校验错误 |
| `frozen_items` 为空 | 任务失败，提示未识别到功能点 |
| 枚举值非法 | 任务失败，提示模型输出不符合契约 |
| 系统字段输出系统编码 | 任务失败，提示必须输出系统中文名 |
| 编号格式或引用关系非法 | 任务失败，提示追溯编号不符合契约 |
| 数据功能缺少支撑过程且无说明 | 任务失败，提示需补充关联过程或复核原因 |
| 出现 Excel 计算字段 | 任务失败，提示模型输出了禁止字段 |
| 复核提示结构非法 | 任务失败，提示 `review_notes` 不符合契约 |

任务处于 `waiting_ai_call` 时，用户可以重新获取 AI 请求包并继续调用模型。用户关闭页面、离开页面或暂未配置 API Key 时，后端不自动判定失败；后续可通过任务保留期或管理员清理策略处理长期未回传任务。

## 15. 可见性与权限

普通用户：

- 可以查看任务状态。
- 可以查看脱敏错误摘要。
- 可以获取后端生成的 AI 请求内容用于前端调用。
- 可以查看最终结果摘要。
- 可以下载 Excel。
- 不在文件列表中看到 `AI请求包.json`、`AI评估.md`、结构化 JSON、Excel payload 或过程 JSON。
- 不下载排查文件。

管理员：

- 可以查看完整任务文件。
- 可以查看 AI 请求包、请求摘要、原始响应、调用错误、AI评估、结构化结果、Excel payload、过程 JSON 和校验错误。
- 可以根据排查信息调整提示词模板、系统知识包、系统场景字典或长度配置。

敏感信息规则：

- API Key 只在用户浏览器本地使用，不写入后端文件。
- 后端不得保存用户 API Key。
- 错误摘要不得包含完整 API Key、鉴权头、Cookie 或其他凭证。

## 16. 与 Excel 生成链路的关系

AI 请求包只服务于 AI 结构化识别，不直接服务于 Excel。

边界如下：

- AI 输出 `AI评估.md` 和 `AI结构化结果.json`。
- 后端校验结构化结果，通过后从 `frozen_items` 生成 Excel payload 的 `items`。
- 后端合并任务配置、页面参数、模板路径、输出路径和目标人天。
- Excel payload 必须保留冻结项中的 `stable_id`、`fact_ids`、`route_ids`、`system_scene_ids`、`linked_process_ids` 和 `linked_data_ids`。
- 脚本基于模板生成 Excel 和 `FPA生成过程.json`。
- 页面展示关键结果时，以脚本产物和后端汇总为准，不以 AI 原始输出为准。

禁止：

- 要求 AI 输出 Excel 公式。
- 要求 AI 输出 Excel 单元格位置。
- 要求 AI 计算人天。
- 要求 AI 判断是否命中目标人天。
- 直接把 AI 输出当成最终 Excel 明细，不经过后端校验和脚本标准化。

## 17. 后续扩展

后续可以扩展：

- 支持更多模型提供方。
- 支持模型接口配置化。
- 支持请求包预览和管理员调试模式。
- 支持更精细的系统资料片段检索。
- 支持多系统需求拆分。
- 支持提示词模板灰度和版本管理。

暂不做：

- 不做后台 Worker 执行链路。`Worker` 是后台任务执行组件通用名称，保留英文。
- 不做 Redis 队列。`Redis` 是第三方组件名称，保留英文。
- 不做资料全文检索。
- 不做提示词版本库。
- 不做自动压缩和自动摘要。
- 不做多模型自动评测。
