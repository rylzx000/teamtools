# TeamTools 整体页面设计

## 1. 设计定位

TeamTools 是轻量内网工具平台，页面首版以 FPA 单模块为主，不展示独立首页、模块卡片或未启用模块预留。

首版所有页面都应遵守同一套平台结构：

- 顶部栏。
- 主工作区。
- 公共错误页。

当前只有 FPA 一个业务模块，首版不展示左侧模块栏。后续确实接入多个模块时，再恢复模块导航能力，避免首版为暂不存在的模块占用空间。

## 2. 页面清单

| 页面 | 建议路径 | 作用 |
|---|---|---|
| 登录页 | `/login` | 用户登录入口 |
| FPA 默认入口 | `/` 或 `/fpa/tasks` | 登录后进入 FPA 任务列表 |
| 无权限页 | `/403` | 无权限访问提示 |
| 未找到页 | `/404` | 路由或资源不存在提示 |

首版登录后直接进入 `/fpa/tasks`。后续新增模块时，再把 `/` 调整为真实模块入口或工作台。

## 3. 平台壳结构

### 3.1 顶部栏

顶部栏主要展示：

- 当前模块标识：`FPA 工作量评估`，使用绿色点作为轻量状态/模块标记。
- 平台名称 `TeamTools` 可保留在浏览器标题或系统标识中，不在首版顶部栏重复展示。
- 当前用户展示名。
- 用户角色标识。
- 退出登录入口。

规则：

- 当前用户信息来自登录态。
- 管理员和普通用户共用同一套顶部栏。
- 退出登录后返回登录页。

### 3.2 模块导航

首版不展示左侧模块栏，只在顶部栏左侧展示当前模块标识 `FPA 工作量评估`。

规则：

- 首版只有 FPA 模块，不展示模块选择栏、首页模块卡片或后续模块预留。
- `任务列表`、`提交评估`、`查看详情` 属于 FPA 模块内页签，放在主工作区顶部。
- 后续确实启用多个模块时，再恢复模块导航；导航只展示已启用模块。
- 普通用户和管理员共用同一模块入口，权限差异体现在数据范围和按钮可见性。

### 3.3 内容区

内容区承载业务页面，建议统一支持以下页面模式：

- 列表页。
- 表单页。
- 详情页。
- 错误页。

通用约定：

- 使用面包屑或页面标题表达当前位置。
- 页面级操作按钮放在标题区右侧。
- 列表、表单和详情页都应保持简洁，不做营销型排版。

## 4. 通用交互规则

- 页面加载失败时显示轻量错误状态，而不是空白页。
- 列表页默认手动刷新，不自动轮询，除非模块页单独约定。
- 表单提交中需要显示忙碌态并禁用重复提交。
- 所有权限按钮都只做前端展示控制，最终权限以后端校验为准。
- 任何页面都不得显示模型 key、环境变量、服务器敏感路径或其他用户数据。

## 5. 通用空态和异常态

建议平台公共页面统一具备以下状态：

- 加载中。
- 无数据。
- 请求失败。
- 无权限。

这些状态尽量复用统一样式，避免每个模块自己发明一套提示样式。
