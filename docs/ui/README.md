# TeamTools 页面设计

## 1. 设计定位

这里记录 TeamTools 平台级页面设计和模块级页面设计。

页面设计和业务设计分开沉淀：

- `docs/ui/` 只记录平台壳、通用页面、布局和交互。
- `docs/modules/` 继续记录各业务模块的需求、流程、接口和数据设计。

首版只有 FPA 一个业务模块，但页面结构仍按多模块平台来设计，后续新增模块时可直接复用壳子。

## 2. 文档列表

- [01-整体页面设计.md](01-整体页面设计.md)：平台壳、登录页、模块入口、通用布局和公共页面规则。
- [modules/fpa-页面设计.md](modules/fpa-页面设计.md)：FPA 模块的提交评估、任务列表、任务详情页面设计。
- [modules/fpa-前端页面样式修改.md](modules/fpa-前端页面样式修改.md)：Open Design 评审后确认的 FPA PC 中后台样式修改原则和原型落地约束。

## 3. 当前边界

本目录不包含以下内容：

- 业务需求说明。
- 数据库设计。
- 接口定义。
- 代码实现。
- 高保真视觉稿。

这里只保留页面结构、交互规则和前端实现时需要遵守的布局约定。
