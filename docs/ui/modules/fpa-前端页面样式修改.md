# FPA 前端页面样式修改

## 1. 文档定位

本文记录 Open Design 原型评审后确认的 TeamTools FPA 页面样式修改原则，供后续工程开发对齐。`Open Design`、`TeamTools`、`FPA`、`API Key`、`DeepSeek`、`HTML/CSS/JS` 保留英文，是因为它们是工具、产品、模块、接口安全字段、模型品牌或技术文件类型的固定名称。

本文只约束页面外观、信息层级和静态交互呈现；业务字段、接口、状态流、权限和文件可见性仍以以下文档为准：

- `docs/ui/modules/fpa-页面设计.md`
- `docs/modules/fpa/02-任务流程设计.md`
- `docs/modules/fpa/03-数据与文件设计.md`
- `docs/modules/fpa/05-接口设计.md`
- `docs/architecture/01-技术架构.md`
- `docs/architecture/03-任务处理与结果生成设计.md`

当前 Open Design 原型快照归档在：

- `docs/ui/prototypes/open-design-teamtools-fpa-current/`

## 2. 总体原则

### 2.1 只做 PC 内网中后台

TeamTools 是轻量内网工具平台，不是移动 App、营销页或大屏。FPA 页面开发时必须保留：

- 顶部栏：平台名、当前模块、用户、角色、退出登录。
- 顶部模块标识：首版固定展示 `FPA 工作量评估`，保留绿色点，不展示左侧模块栏。
- 主工作区：承载模块页面，不把整个平台做成单页业务应用。
- FPA 页签：`任务列表`、`提交评估`、`查看详情` 常驻展示。
- panel/card/table 风格：轻边框、轻背景、少阴影、信息密度优先。

首版按 PC 端 1366px 宽度优先设计。常规桌面宽度下保持顶部模块标识、FPA 页签和主工作区稳定，不做移动端单列化。

### 2.2 保留现有视觉，不重做风格

当前选定方向为清爽现代的中后台风格：

- 主色使用浅青绿色系，作为按钮、选中态和少量状态强调。
- 背景使用浅灰蓝工作区，内容面板使用白色或近白色。
- 边框使用浅灰细线，不使用重阴影和装饰背景。
- 字号、间距和圆角以提交评估页当前版本为基准，三页切换时视觉节奏要一致。
- 不使用营销式大标题、夸张渐变、大面积插画、emoji 图标或宣传文案。

### 2.3 三页切换要丝滑

三个 FPA 页签切换时，应保持同一套页面节奏：

- 页签位置、高度、下方间距一致。
- 页面标题字号一致。
- 标题下横线位置一致。
- 首个内容面板距离标题区一致。
- panel 内边距、表格字号、按钮高度、标签字号一致。

如果新增页面局部样式，应优先抽成同一套 FPA 页面级规则，避免某一页单独松散或拥挤。

## 3. 任务列表页原则

### 3.1 直接展示列表

任务列表页是 FPA 模块默认工作页，不需要解释性文案。页面应直接进入任务列表，不展示以下内容：

- `FPA MODULE`
- `FPA 任务`
- `查看评估任务状态、下载产物...` 这类说明语

保留一个短标题 `任务列表` 即可。

### 3.2 操作与筛选合并

任务列表页不需要页头主按钮区。按钮组织方式：

- 状态筛选：`全部`、`进行中`、`已完成`、`失败`、`已取消`。
- `手动刷新` 放在状态筛选同一行。
- `提交评估` 不在任务列表页头重复出现；用户可通过 FPA 页签进入提交页。

### 3.3 表格保持高信息密度

表格是任务列表页的主体，应优先保证 1366px 宽度下信息清晰：

- 状态使用后端 `status_label` 展示。
- 进行中状态包含：`等待AI调用`、`结果校验中`、`生成结果中`。
- 列表操作保留：`查看详情`、`下载 Excel`、`取消`、`复制并重新生成`。
- 管理员额外字段可以展示提交人、失败阶段，但不要挤压普通用户核心字段。

## 4. 提交评估页原则

### 4.1 一屏优先

提交评估页功能不多，应尽量在 1366px 宽度下一屏内完成主要填写。空间策略：

- 标题区保持短，不放长说明。
- 顶部字段使用一行紧凑布局。
- `需求名称` 单独一行，宽度与 Markdown 粘贴框一致。
- `粘贴 Markdown 内容` 文本框固定高度，输入内容多时在文本框内部滚动。
- 上传 `.md` 文件区压缩高度，但保留点击和拖拽上传能力。
- 规则提示只保留用户提交前真正需要知道的限制。

### 4.2 字段布局

提交页当前字段顺序应保持：

1. 第一行短字段：系统选择、规模计数时机、完整性级别、目标人天。
2. 第二行：需求名称，单独一行。
3. 主输入区：Markdown 粘贴框和 `.md` 上传。
4. 右侧或提交动作附近：API Key 输入和本地保存提示。

字段规则：

- `需求名称` 可选，为空时由后端自动生成。
- `规模计数时机` 展示带系数的选项：`1.39 估算早期`、`1.21 估算中期`、`1.10 估算晚期`。
- `完整性级别` 与规模计数时机同排展示，便于用户提交前统一确认估算口径。
- `目标人天` 放最后，输入框更窄并右对齐。

### 4.3 API Key 位置

API Key 不是业务输入主体，不应占用主表单区域。建议放在提交按钮下方或提交动作附近：

- 显示 `DeepSeek · deepseek-v4-flash` 的简短说明。
- API Key 使用密码输入框。
- `记住到本机浏览器` 使用 checkbox。
- 说明文案保持短：仅本机保存，不上传后端。

提交后应自动进入查看详情页。若本机已有 API Key，详情页可自动模拟模型调用；若没有 API Key，详情页停在等待处理状态。

## 5. 查看详情页原则

### 5.1 默认展示最新任务

`查看详情` 页签要常驻展示。用户不是从某条任务跳转过来时，静态原型默认展示最新任务详情；工程实现可按路由参数或最近任务接口决定默认任务。

页面标题使用 `查看详情`，任务元信息放在标题横线下方，例如：

```text
交易审批流程改造 · 任务 ID：TASK-202606-014 · 资金系统 · 06-27 09:12
```

### 5.2 保留必要信息，删除重复信息

查看详情页不应默认展示过多技术过程。普通用户打开后优先看到：

- 状态进度。
- 任务摘要。
- 结果查看。
- 必要的失败提示。

不要默认常驻展示大块 `AI 调用区`、模型供应商、模型名称、AI 请求包状态、API Key 状态和回传按钮。它们对普通用户正常流程意义不大，只在异常或管理员排查时需要。

### 5.3 状态进度 + 任务摘要同排

查看详情页顶部工作区建议一行两列：

- 左侧：状态进度。
- 右侧：任务摘要。

任务摘要使用竖版卡片，但每一行标题和内容必须在同一行展示，不要拆成上下两行。状态进度每一步的标题尽量保持一行展示，最后一步也不要换行。

任务摘要不重复展示当前状态；状态只由左侧状态进度表达。任务摘要保留目标人天、结果中值、是否命中目标、耗时和 Excel 下载入口。

状态进度使用当前流程：

1. 任务创建：接收任务和输入内容。
2. AI 请求包已生成：后端已生成模型请求内容。
3. 等待AI调用：浏览器使用本地 API Key 调用模型。
4. 结果校验中：后端校验 AI 结构化 JSON。
5. 生成结果中：后端写入 Excel 模板。
6. 已完成 / 失败 / 已取消。

列表页状态列与详情页状态进度必须使用同一个后端状态源，关系如下：

| 后端 `status` | 列表页 `status_label` | 详情页状态进度 |
|---|---|---|
| `waiting_ai_call` | 等待AI调用 | 高亮“等待AI调用” |
| `validating_result` | 结果校验中 | 高亮“结果校验中” |
| `generating_result` | 生成结果中 | 高亮“生成结果中” |
| `completed` | 已完成 | 高亮“已完成” |
| `failed` | 失败 | 高亮“失败”，失败阶段在详情失败区展示 |
| `cancelled` | 已取消 | 高亮“已取消” |

`进行中` 只是列表筛选分组，不是页面展示状态；包含 `waiting_ai_call`、`validating_result` 和 `generating_result`。`失败阶段` 不是状态列，不在任务列表常驻展示。

### 4.4 列表操作列收敛

任务列表操作列应保持稳定、短文案、单行展示：

- 默认展示 `查看`。
- 仅当任务已完成且可下载时额外展示 `下载`。
- 不在列表展示 `取消任务`、`复制并重新生成`、`继续处理` 等多按钮操作。
- 低频或异常操作进入查看详情页的 `任务操作` 折叠区。
- 1366px 宽度下列表不应横向滚动；系统列、操作列应一行展示，不截断按钮文案。

### 5.4 模型调用信息只做异常兜底

模型调用状态不是完全没用，但不应作为默认模块展示。它的用途只有：

- 未配置 API Key 时，提示需要配置后继续。
- 模型调用失败时，提供重新调用入口。
- 管理员排查时查看模型、请求摘要、调用错误和日志。

因此普通状态下只通过状态进度表达处理阶段；`调用模型`、`重新调用`、`回传结果` 等静态原型按钮应放在默认折叠的手动处理区域或管理员/失败排查区。

### 5.5 结果查看优先

详情页默认可见的结果区应聚焦用户产物：

- Excel 下载入口放在右侧任务摘要区，未生成时展示不可用状态。
- 结果查看区只保留预览类产物：`AI分析.md` 和 `FPA生成过程.json`。
- `AI分析.md`：折叠预览、复制，不下载。
- `FPA生成过程.json`：折叠预览、复制，不下载。

管理员额外可见内容放在默认折叠区：

- `AI请求摘要.json`
- `AI结构化结果.json`
- `AI调用错误.json`
- `task.log`
- JSON 校验错误、异常堆栈摘要、工作目录相对路径。

不要把 `AI请求包.json` 放进普通文件列表，也不要提供下载入口。

### 5.6 质量与校验默认折叠

`模型调用次数`、`无资料模式`、`资源可读取`、`目标异常`、`条目过多`、`AI 调用失败`、`JSON 校验失败` 等信息与状态进度有重叠，且多为排查信息。默认不要占据详情页主要区域，应放入 `管理员与失败排查` 折叠项。

## 6. 安全与权限边界

页面原型和正式实现都不能向普通用户展示：

- 模型 API Key。
- 服务器敏感路径。
- 完整源码路径。
- 环境变量。
- 完整日志。
- 其他用户任务内容。

普通用户能看到自己的任务和脱敏错误；管理员可以看到排查摘要和必要日志入口，但仍不应在页面上明文展示敏感密钥。

## 7. 原型转开发说明

当前 Open Design HTML 原型用于确认 PC 中后台视觉、页面密度和交互位置，不应被开发实现逐字照搬。开发时必须把原型里的演示数据、解释性文案和模拟按钮替换为真实接口数据、真实状态和真实权限控制。

### 7.1 总体转换规则

- HTML/CSS/JS 原型只作为视觉结构参考，正式字段、状态、接口、权限和文件规则以 `docs/modules/fpa/`、`docs/ui/modules/fpa-页面设计.md` 和 OpenSpec 规格为准。
- 原型中的示例任务名称、任务 ID、系统名、时间、目标人天、结果中值和状态值，开发时全部接接口返回，不写死在页面里。
- 原型中的“默认收起”“仅排错时展开”“正常流程无需操作”“等待后端生成”等说明性文案，正式页面应删除、替换为真实数据，或只作为 tooltip / 帮助文档使用。
- 静态原型中的模拟按钮和模拟状态切换只用于说明流程，不代表正式产品必须保留手动操作入口。
- 如果原型与接口文档、数据文档或任务流程文档冲突，以工程文档为准；不要为了贴合原型而改业务规则。

### 7.2 页面级处理清单

| 页面 | 原型内容 | 开发处理 |
|---|---|---|
| `index.html` | 旧首页模块卡片和后续模块预留 | 正式开发删除；登录成功后直接进入 FPA 任务列表。 |
| `fpa-tasks.html` | 示例任务行、示例状态 | 接任务列表接口；状态列直接展示 `status_label`，详情页进度按 `status` 映射高亮。 |
| `fpa-tasks.html` | 管理员字段开关与脚注说明 | 正式开发按登录角色控制字段可见性，不在普通用户页面展示设计说明。 |
| `fpa-submit.html` | 表单占位文案、上传限制、提交规则 | 保留必要输入提示和校验提示；不要把流程解释写成大段说明。 |
| `fpa-submit.html` | API Key 本地保存模拟逻辑 | 正式实现只保存在本机浏览器，不上传后端；具体存储策略按安全评审执行。 |
| `fpa-detail.html` | 示例任务摘要、状态进度、结果预览 | 接任务详情接口和产物接口；无数据时展示真实空状态。 |
| `fpa-detail.html` | 状态步骤后的解释文案 | 默认删除；如确需解释，放在 tooltip、帮助入口或文档中。 |
| `fpa-detail.html` | 手动处理、调用模型、回传结果模拟按钮 | 正常流程不常驻展示；仅无 API Key、调用失败或管理员排查场景展示兜底入口。 |
| `fpa-detail.html` | 管理员与失败排查示例内容 | 仅管理员可见，并替换为真实脱敏排查数据。 |
| `fpa-detail.html` | 安全边界说明文案 | 不作为普通用户页面内容展示；规则保留在工程文档和权限实现中。 |

### 7.3 正式页面应删除或替换的内容

正式开发时重点清理以下原型内容：

- 首页模块卡片、后续模块预留、左侧模块栏、左上角 `TT / TeamTools / 内部工具平台` 品牌锁定区。
- 顶部栏里的 `TeamTools / FPA 工作量评估` 面包屑；顶部仅保留带绿点的 `FPA 工作量评估` 模块标识。
- 状态步骤后面的解释句，例如“后端已生成模型请求内容”“浏览器使用本地 API Key 调用模型”。
- 原型说明句，例如“默认收起”“仅排错时展开”“正常流程无需操作”。
- 示例任务数据，例如 `TASK-202606-014`、示例系统、示例时间、示例任务名称。
- 占位产物说明，例如“等待后端生成”“暂无内容”“完成后自动补齐”。
- 普通用户不该看到的权限说明、安全说明、管理员排查说明。
- 静态演示用的回传按钮、状态切换按钮和模拟提示。

### 7.4 正式页面必须保留的内容

以下内容不是“说明性废话”，开发时应保留或接真实数据：

- 平台壳子：顶部栏、顶部模块标识、主工作区。
- FPA 页签：任务列表、提交评估、查看详情。
- 表单字段：系统选择、规模计数时机、完整性级别、目标人天、需求名称、Markdown 内容、`.md` 文件上传、API Key 配置。
- 任务列表字段：任务名称、系统、状态、提交时间、完成时间、目标人天、结果中值、是否命中目标、操作；管理员额外展示提交人，不常驻展示失败阶段。
- 详情页核心区：状态进度、任务摘要、结果查看、失败提示、管理员排查折叠区。
- 产物规则：Excel 下载入口位于任务摘要；`AI分析.md` 和 `FPA生成过程.json` 位于结果查看区，可预览/复制，不对普通用户提供下载。

### 7.5 页面字段来源

正式开发时按以下来源替换静态原型内容：

| 页面/区域 | 字段或内容 | 正式来源 |
|---|---|---|
| 登录后入口 | 默认页面 | 登录成功后直接进入 FPA 任务列表，不展示首页模块卡片。 |
| 顶部模块标识 | 当前模块 | 首版固定展示 `FPA 工作量评估`，保留绿色点；不展示左侧模块栏。 |
| 顶部栏 | 当前用户、角色、退出登录 | 登录态和权限信息。 |
| 提交评估 | 系统选择 | 后端提交页配置接口，不在前端写死。 |
| 提交评估 | 规模计数时机 | 后端提交页配置接口，默认值以后端返回为准。 |
| 提交评估 | 完整性级别 | 后端提交页配置接口，默认值以后端返回为准。 |
| 提交评估 | 需求名称、目标人天、Markdown、上传文件、API Key | 用户输入；API Key 只保存本机浏览器，不上传后端。 |
| 任务列表 | 任务行和状态 | 任务列表接口；状态列展示 `status_label`。 |
| 查看详情 | 任务摘要、状态进度、产物、失败信息 | 任务详情接口和产物接口；详情进度按 `status` 映射。 |

```text
5. 开发时把静态数据替换为接口字段，不新增营销文案或解释性模块。
```

## 8. 开发落地建议

- 先实现 PC 端 1366px 主场景，再考虑更窄窗口的降级展示。
- 样式实现优先复用现有类：`app-shell`、`topbar`、`module-badge`、`workspace`、`fpa-tabs`、`panel`、`table-panel`、`summary-strip`、`button`。
- 表格、表单、详情三类页面统一字号和间距，不为单页重新发明一套局部风格。
- 静态原型里的模拟交互只作为开发参考，不代表需要在正式前端中保留所有演示按钮。
- 若原型与工程文档冲突，以工程文档、接口设计和 OpenSpec 规格为准。
