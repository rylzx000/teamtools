# FPA 模块页面设计

## 1. 设计定位

本文件只记录 FPA 模块内部页面设计。平台级顶部栏、首版单模块入口、登录态和公共错误页由 [整体页面设计](../01-整体页面设计.md) 统一定义。

FPA 页面基于现有 Open Design 原型的大框架继续演进：

- 顶部模块标识：`FPA 工作量评估`，保留绿色点。
- 顶部用户信息区。
- 主内容工作区。
- FPA 模块页签。
- 卡片式表单、表格、状态进度、结果产物和失败信息区。

Open Design 原型中的布局骨架可以保留，但页面文案、状态、产物和按钮逻辑必须按当前架构调整。

当前 FPA 主链路：

```text
提交评估
-> 后端创建任务并生成 AI 请求包
-> 任务详情页展示等待AI调用
-> 前端获取 AI 请求包并调用外部模型
-> 前端回传 AI 结果或调用错误
-> 后端校验 JSON 并生成 Excel
-> 用户下载 Excel
```

## 2. 页面清单

FPA 首版只覆盖三个页面：

| 页面 | 建议路径 | 作用 |
|---|---|---|
| 任务列表 | `/fpa/tasks` | FPA 模块默认页，查看历史任务和进入详情 |
| 提交评估 | `/fpa/submit` | 新建 FPA 评估任务 |
| 任务详情 | `/fpa/tasks/:id` | 查看任务状态、执行模型调用、查看结果和下载 Excel |

页面框架沿用 Open Design：

- 登录成功后默认进入任务列表页，不再展示单独首页、模块卡片或后续模块预留。
- 首版只有 FPA 模块，不展示左侧模块栏；顶部左侧固定展示 `FPA 工作量评估`，保留绿色点。
- 顶部栏只展示当前模块标识、当前用户、角色和退出登录，不再显示 `TeamTools / FPA 工作量评估` 面包屑。
- FPA 模块内使用页签切换 `任务列表`、`提交评估`、`任务详情`。
- 内容区使用 `page-title-row`、`panel`、`summary-strip`、`table-panel` 等结构。

## 3. 提交评估页

### 3.1 页面结构

沿用 Open Design 的提交页结构，从上到下分为：

1. FPA 模块页签区。
2. 基本信息区：系统、规模计数时机、完整性级别、目标人天、需求名称。
3. 需求输入区：粘贴 Markdown 内容、上传 `.md` 文件。
4. 模型配置提示区：模型供应商、模型名称、API Key 本地使用说明。
5. 校验提示区：字符数、文件大小、输入要求。
6. 操作区：提交。

说明：

- 提交页负责创建任务，不直接生成 Excel。
- 提交成功后跳转任务详情页，由详情页进入模型调用流程。
- 页面可以提示“后端会生成 AI 请求包，浏览器使用本地 API Key 调用模型”。

### 3.2 表单字段

| 字段 | 控件 | 是否必填 | 规则 |
|---|---|---:|---|
| 系统 | 下拉选择 | 是 | 来自后端配置，不在前端写死；每次只能选择一个系统 |
| 需求名称 | 单行输入 | 否 | 为空时由后端生成短名称或兜底名称 |
| 目标人天 | 数字输入 | 否 | 大于 0；最多 1 位小数；异常只提示风险，不阻止提交 |
| 规模计数时机 | 下拉选择 | 否 | 来自后端配置；默认值以后端返回为准 |
| 完整性级别 | 下拉选择 | 否 | 来自后端配置；用于完整性调整因子 |
| 粘贴内容 | Markdown 文本框 | 否 | 与上传文件至少提供一项；有效内容不超过 2 万字符 |
| 上传文件 | 单文件上传 | 否 | 只允许 `.md`；文件大小不超过 256KB；有效内容不超过 2 万字符 |

Open Design 原型调整点：

- `需求名称` 从必填改为可选。
- 新增 `规模计数时机` 和 `完整性级别` 字段。
- `系统选择`、`规模计数时机`、`完整性级别` 都必须来自后端配置，不进入正式实现硬编码。
- Open Design 静态原型可使用 JS 模拟后端配置加载，但正式开发必须替换为后端配置接口。

### 3.3 模型配置提示

提交页可展示轻量模型配置提示，但不负责真正调用模型。

建议展示：

| 字段 | 控件 | 说明 |
|---|---|---|
| 模型供应商 | 下拉或只读文本 | 首版默认 DeepSeek，可后续扩展 |
| 模型名称 | 输入或只读文本 | 默认 `deepseek-v4-flash`，以后可配置 |
| API Key | 密码输入 | 用户个人 Key，仅浏览器本地使用 |
| 本地保存 | 勾选框 | 是否保存到浏览器本地；默认不上传后端 |

安全要求：

- 团队公共模型 Key 不写入前端代码。
- 用户个人 API Key 不上传后端。
- 页面不明文展示已保存 Key，只显示已配置状态。
- 调用模型前如果没有 Key，详情页需要提示用户补充。

### 3.4 交互规则

- 系统未选择时，提交按钮不可用。
- 粘贴内容和上传文件都为空时，提交按钮不可用。
- 粘贴内容显示有效字符计数。
- 上传文件后展示文件名、文件大小和移除按钮。
- 同时提供粘贴内容和上传文件时，提示后端会按“粘贴内容在前、上传文件在后”合并。
- 目标人天格式错误时在字段下方提示。
- 提交中禁用表单和提交按钮，按钮显示处理中状态。
- 提交成功后跳转到任务详情页。

提交成功后，详情页应展示：

```text
任务已创建
AI 请求包已生成
等待浏览器调用模型
```

### 3.5 复制并重新生成

从历史任务点击“复制并重新生成”时，推荐进入提交评估页，并预填：

- 系统。
- 需求名称。
- 目标人天。
- 规模计数时机。
- 原任务合并输入内容。

用户修改后再提交，生成新的任务记录和结果文件，不覆盖原任务。

注意：

- `POST /api/fpa/tasks/{id}/rerun` 更适合“基于原输入直接新建任务”。
- 如果支持“复制后修改再提交”，前端需要获取原任务合并输入内容。
- 原任务不被覆盖，新任务通过 `rerun_from_task_id` 关联旧任务。

## 4. 任务列表页

### 4.1 页面结构

沿用 Open Design 的任务列表页结构：

1. 页面标题区：`任务列表`。
2. 筛选与刷新区：状态筛选和手动刷新放在同一工具条。
3. 表格区：任务列表。
4. 分页区：页码、每页数量、总数。

任务列表页不自动刷新，只提供手动刷新按钮。

### 4.2 普通用户列表字段

| 字段 | 说明 |
|---|---|
| 任务名称 | 点击进入任务详情 |
| 系统 | 展示系统名称 |
| 状态 | 使用后端返回的 `status_label` 展示 |
| 提交时间 | 对应 `created_at` |
| 完成时间 | 未完成时显示 `-` |
| 目标人天 | 未填写时显示 `-` |
| 结果中值 | 仅完成后展示 |
| 是否命中目标 | 命中、未命中、未设置目标 |
| 操作 | 列表只展示查看；已完成且可下载时额外展示下载 |

### 4.3 管理员额外字段

管理员列表额外展示：

- 提交人。

普通用户只能看到自己提交的任务。管理员可以看到全部 FPA 任务。

失败阶段不作为任务列表常驻列展示。列表只保留 `状态` 列；失败阶段、脱敏原因和建议操作进入任务详情页的失败区或管理员排查区查看。

### 4.4 状态筛选

状态筛选保留 Open Design 的轻量筛选方式：

| 筛选项 | 包含状态 |
|---|---|
| 全部 | 所有任务 |
| 进行中 | `waiting_ai_call`、`validating_result`、`generating_result` |
| 已完成 | `completed` |
| 失败 | `failed` |
| 已取消 | `cancelled` |

状态展示规则：

- API 字段使用枚举值，例如 `waiting_ai_call`。
- 页面展示使用 `status_label`，例如 `等待AI调用`。
- 不在前端写死旧状态文案。

### 4.5 操作规则

- 列表操作列保持轻量：默认只展示“查看”。
- “下载”仅在任务已完成且 `can_download_excel = true` 时额外展示。
- “取消任务”和“复制并重新生成”不放在列表常驻操作列，进入查看详情页后在任务操作区展示。
- 所有按钮状态以后端返回字段为展示依据，真正权限以后端接口校验为准。

Open Design 原型调整点：

- 示例状态 `AI 分析中` 改为 `等待AI调用` 或 `结果校验中`。
- 示例失败阶段 `解析结果中` 改为 `结果校验中` 或具体后端失败阶段。

## 5. 任务详情页

### 5.1 页面结构

沿用 Open Design 的详情页结构，从上到下分为：

1. 任务头部。
2. 摘要条。
3. 状态进度。
4. AI 调用区。
5. 结果摘要。
6. 质量提示。
7. 失败信息。
8. 结果产物。
9. 管理员排查信息。

### 5.2 任务头部

展示内容：

- 任务名称。
- 任务状态。
- 任务 ID。
- 系统名称。
- 提交人，管理员可见。
- 提交时间。
- 页面级操作入口：任务操作折叠区承载取消任务、复制并重新生成等低频动作；Excel 下载入口放在任务摘要区。

### 5.3 任务摘要

任务摘要与状态进度同排展示，使用 Open Design 的右侧竖版摘要风格。

展示字段：

- 目标人天。
- 结果中值。
- 是否命中目标。
- 耗时。
- Excel 下载入口。

`当前状态` 不在任务摘要中重复展示，只由状态进度表达。未生成结果时，结果字段显示 `-` 或“等待生成”；Excel 未生成时展示不可用下载状态。

### 5.4 状态进度

状态集合：

| API 状态 | 页面展示 |
|---|---|
| `draft` | 草稿 |
| `waiting_ai_call` | 等待AI调用 |
| `validating_result` | 结果校验中 |
| `generating_result` | 生成结果中 |
| `completed` | 已完成 |
| `failed` | 失败 |
| `cancelled` | 已取消 |

列表页和详情页使用同一个后端状态源：

- 列表页直接展示当前任务的 `status_label`，只表达“当前在哪一步”。
- 查看详情页使用 `status` 映射并高亮状态进度中的对应节点。
- `失败阶段` 不是任务状态本身，只是失败任务的补充排查字段，不进入列表列。

页面进度建议展示：

```text
任务创建
AI请求包已生成
等待AI调用
结果校验中
生成结果中
已完成 / 失败 / 已取消
```

Open Design 原型调整点：

- 移除 `读取资料中`。
- 移除 `AI 分析中`。
- 移除 `解析结果中`。
- `生成 Excel 中` 可作为 `生成结果中` 的页面说明，但 API 状态以 `generating_result` 为准。

轮询规则：

- 执行中状态每 2-3 秒轮询一次 `GET /api/fpa/tasks/{id}`。
- 进入已完成、失败或已取消后停止轮询。
- 已取消状态展示取消时间和取消人，取消人字段按权限展示。

### 5.5 AI 调用区

AI 调用区是当前方案相对 Open Design 原型必须新增的核心区域。

展示条件：

- 任务状态为 `waiting_ai_call`。
- 后端返回 `can_fetch_ai_request = true` 或 `can_submit_ai_result = true`。

区域内容：

| 内容 | 说明 |
|---|---|
| AI 请求包状态 | 已生成、已获取、获取失败 |
| 模型供应商 | 默认 DeepSeek，可后续配置 |
| 模型名称 | 默认 `deepseek-v4-flash` |
| API Key 状态 | 未配置、已在浏览器本地配置 |
| 调用按钮 | 获取请求包并调用模型 |
| 重试按钮 | 前端调用失败后允许重试 |
| 回传状态 | 待回传、回传中、已回传、回传失败 |

交互规则：

- 点击“调用模型”时，前端先请求 `GET /api/fpa/tasks/{id}/ai-request`。
- 前端使用浏览器本地 API Key 调用外部模型。
- 调用成功后，通过 `POST /api/fpa/tasks/{id}/ai-result` 回传结果。
- 调用失败时，通过 `POST /api/fpa/tasks/{id}/ai-result` 回传脱敏错误，或允许用户前端重试。
- 前端调用期间可以展示本地进度，但后端状态仍以任务详情接口为准。
- 如果用户离开页面，前端应提示模型调用可能中断。

取消规则：

- 后端取消只改变任务状态。
- 如果浏览器正在调用模型，前端需要自行中止请求或提示用户关闭等待。

### 5.6 结果摘要

展示字段：

- 目标人天。
- 结果中值。
- 是否命中目标。
- 耗时。
- 前端模型调用重试次数或重新提交次数。
- 是否无资料模式。

Open Design 原型调整点：

- 移除 `模板版本`。
- 移除 `资料版本`。
- MVP 不展示版本追溯信息。

### 5.7 质量提示

质量提示只提示风险，不阻止用户下载和使用结果。

提示项包括：

- 是否无资料模式。
- 是否目标人天异常。
- 是否功能点条目过多。
- 是否发生过 AI 调用失败。
- 是否发生过 JSON 校验失败。
- 当前资源文件是否可读取。

Open Design 原型调整点：

- `JSON 修复重试` 改为 `JSON 校验失败`。
- `模板版本` 改为 `资源文件可读取`。
- `资料版本` 改为 `系统资料可读取` 或直接不展示。

### 5.8 失败信息

失败任务展示：

- 失败阶段。
- 脱敏后的错误原因。
- 建议操作。
- 复制并重新生成或重新运行入口。

普通用户不能看到：

- 模型 API Key。
- 服务器敏感路径。
- 完整源码路径。
- 环境变量。
- 完整日志。
- 其他用户任务内容。

管理员额外可见：

- AI 请求摘要。
- AI 原始响应。
- AI 调用错误。
- JSON 校验错误。
- 完整任务日志入口。
- 异常堆栈摘要。
- 工作目录相对路径或受控展示路径。

### 5.9 结果产物

普通用户成功任务展示：

| 产物 | 页面能力 | 下载能力 |
|---|---|---|
| Excel 文件 | 在任务摘要区展示下载入口 | 可下载 |
| `AI分析.md` / `AI评估.md` | 不展示 | 不提供下载 |
| `FPA生成过程.json` | 不展示 | 不提供下载 |

管理员额外可见：

| 产物 | 页面能力 | 下载能力 |
|---|---|---|
| `AI请求摘要.json` | 折叠预览 | 不提供普通下载 |
| `AI结构化结果.json` | 折叠预览 | 不提供普通下载 |
| `AI调用错误.json` | 失败时展示 | 不提供普通下载 |
| `task.log` | 日志入口或摘要 | 不提供普通下载 |

说明：

- `AI请求包.json` 不在普通文件列表展示。
- 普通用户通过 `ai-request` 专用接口获取请求包，不作为产物下载。
- Excel 是唯一首版下载产物，下载入口放在任务摘要区，不在结果查看区重复展示。
- 结果查看区仅管理员可见，用于预览和复制 `AI分析.md` / `AI评估.md` 及必要排查产物。
- 普通用户不查看、复制或下载过程文件；Excel 是普通用户唯一下载产物。

## 6. 按钮状态和权限规则

| 按钮 | 可用条件 | 说明 |
|---|---|---|
| 提交评估 | 表单校验通过 | 请求中禁用 |
| 手动刷新 | 任务列表页可见 | 请求中禁用 |
| 查看详情 | 用户有访问权限 | 普通用户只看自己的任务 |
| 调用模型 | `can_fetch_ai_request = true` | 获取 AI 请求包并在浏览器侧调用模型 |
| 回传 AI 结果 | `can_submit_ai_result = true` | 将模型响应或调用错误回传后端 |
| 取消任务 | `can_cancel = true` | 点击后需要确认 |
| 下载 Excel | `can_download_excel = true` | 仅下载 Excel，不下载过程文件 |
| 复制并重新生成 | 原输入可用且有访问权限 | 推荐进入提交页预填后再提交 |
| 直接重新运行 | 后端允许且用户确认 | 使用 `/rerun` 直接创建新任务 |
| 复制 AI 分析 | 管理员且 `ai_analysis_md.available = true` | 复制页面内容 |
| 复制生成过程 JSON | 管理员且 `fpa_process_json.available = true` | 复制页面内容 |

前端只负责按钮展示、禁用和确认提示。所有关键操作都必须由后端重新校验权限和状态。

## 7. 前端开发说明

### 7.1 页面组织

FPA 相关代码建议独立组织：

- FPA API 客户端。
- FPA 页面组件。
- FPA 表单校验。
- FPA 状态标签。
- FPA 任务表格。
- FPA AI 调用区。
- FPA 详情产物预览。

### 7.2 刷新和轮询

- 任务列表页只手动刷新。
- 任务详情页仅在执行中状态轮询。
- 轮询间隔 2-3 秒。
- 终态停止轮询。
- 页面离开时停止轮询。

执行中状态包括：

- `waiting_ai_call`
- `validating_result`
- `generating_result`

### 7.3 模型调用安全

- 用户个人 API Key 只在浏览器本地使用。
- 不把用户个人 API Key 上传后端。
- 不把团队公共模型 Key 写死到前端代码。
- 不在页面明文展示已保存 API Key。
- 调用错误回传后端前需要脱敏。
- 不在日志、事件、产物中记录完整请求头。

### 7.4 安全展示

前端不得展示：

- 模型 API Key。
- 环境变量。
- 服务器敏感绝对路径。
- 完整源码路径。
- 其他用户任务内容。
- 不允许下载的过程文件。

管理员可见的排查信息也应以后端脱敏和权限校验结果为准。

## 8. Open Design 原型调整清单
