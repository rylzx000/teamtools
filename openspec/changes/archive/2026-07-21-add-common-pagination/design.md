# 公共分页能力设计

## 后端设计

- 新增 `backend/app/pagination.py`，集中处理 `page`、`page_size` 解析、最大值限制、`limit/offset` 计算和统一响应字段。
- FPA 任务列表继续按创建时间倒序查询，新增 `COUNT(*)` 获取总数，并用公共分页响应包装。
- 管理员额度列表按管理员优先、用户名和展示名稳定排序；分页前仍按需要懒初始化当前页用户额度。

## 前端设计

- 在 `frontend/src/App.tsx` 内新增共享分页类型、状态 helper 和 `PaginationBar` 组件，符合当前单文件前端结构。
- 任务列表页和模型配置页共用分页控件、每页条数选项和页码切换逻辑。
- 切换筛选或每页条数时回到第 1 页；保存或重置额度后刷新当前页。

## 测试策略

- 后端测试覆盖默认分页、页大小上限、响应元数据、任务排序、额度排序和普通用户权限。
- 前端以 `npm run build` 覆盖 TypeScript 和构建可用性。
