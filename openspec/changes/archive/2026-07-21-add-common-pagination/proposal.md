# 公共分页能力补充

## 背景

团队公用模型 Key 与额度能力已完成首轮落地。当前 FPA 任务列表和管理员用户额度列表仍一次性返回全量数据，随着任务和用户增长会影响页面加载和表格可用性。

## 目标

- 为后端提供统一分页参数校验、`LIMIT/OFFSET` 计算和响应结构。
- 将 FPA 任务列表和管理员用户额度列表接入统一分页协议。
- 前端复用同一套分页控件，保持现有列表样式和操作能力。

## 非目标

- 不做游标分页、无限滚动、复杂排序框架或搜索筛选。
- 不改变 FPA 任务状态机、公用 Key 扣减规则、模型调用流程或分页以外的业务能力。

## 风险

- 列表接口返回结构变化会影响前端读取 `items`、`total` 的位置。
- 额度表分页后，单人保存、单人重置和批量操作需要刷新当前页并保持分页状态。
